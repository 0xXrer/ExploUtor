# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |
| < 1.0   | :x:                |

Only main branch is actively maintained.

## Reporting a Vulnerability

**Contact:**
- Discord: Mention `@elowen.im` on https://discord.gg/Kc8Gt8apUD (don't DM, I might not respond)
- Telegram: @ImXrer
- GitHub Security Advisory (preferred for sensitive issues)

**Response time:**
- Acknowledgment: 24-48 hours
- Critical fix: 7 days
- High severity: 14 days
- Medium/Low: when available

**What to report:**
- RCE in ExploUtor itself
- Memory corruption in core
- Security mechanism bypasses
- Credential leaks

**Not a bug:**
- "Works too well" is a feature
- AV detection is expected for security tools
- Exploits in target systems are literally the point

**Disclosure:**
- Coordinated disclosure
- Public after fix
- CVE if warranted

**Recognition:**
- Shoutouts in releases
- Contributors badge
- Respect

## Scope

**In scope:**
- ExploUtor framework
- Core modules
- API/CLI interfaces

**Out of scope:**
- Exploits in target systems (that's the feature)
- Social engineering vectors
- Security tool detection
- Dependencies (report upstream)
